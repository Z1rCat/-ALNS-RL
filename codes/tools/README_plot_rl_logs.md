# RL 日志绘图说明

本目录的 `plot_rl_logs.py` 会遍历 `codes/logs/run_*` 文件夹，并在每个运行目录内生成多张图片。下面说明每张图的横纵坐标含义和解读方式。

## 1. train_reward_curve.png
- 横轴：`step_idx`（全局交互步数）。
- 纵轴：`reward`（训练阶段单步奖励，通常为 0/1）。
- 解读：曲线越高表示训练阶段获得的奖励越多；若长期接近 0，说明策略在训练阶段效果不佳。

## 2. eval_reward_curve.png
- 横轴：`step_idx`（评估时刻对应的全局步数）。
- 纵轴：`avg_reward`（评估平均奖励）。
- 解读：用于观察评估性能随训练进度的变化；阴影（若有）为 `std_reward` 的波动区间。

## 3. implement_reward_curve.png
- 横轴：`step_idx`（实施阶段交互步数）。
- 纵轴：`reward`（实施阶段单步奖励，通常为 0/1）。
- 解读：实施阶段的真实执行效果曲线；越高说明行动更“有效”。

## 4. action_dist_removal.png
- 左图：训练阶段；右图：实施阶段。
- 横轴：动作编号（0=等待/保持，1=重新规划）。
- 纵轴：动作出现次数（count）。
- 解读：观察移除阶段策略偏好。左侧（train）用于训练习惯，右侧（implement）用于实际执行偏好。

## 5. action_dist_insertion.png
- 左图：训练阶段；右图：实施阶段。
- 横轴：动作编号（0=接受插入，1=拒绝插入）。
- 纵轴：动作出现次数（count）。
- 解读：观察插入阶段策略偏好。注意插入阶段的 0/1 语义与移除阶段不同。

## 6. stage_counts.png
- 横轴：`stage`（例如 begin_removal、send_action、finish_insertion 等）。
- 纵轴：出现次数（count）。
- 解读：用于检查日志中各阶段事件的覆盖度与频次，确认是否缺失某个阶段。

## 7. reward_hist.png
- 左图：训练阶段；右图：实施阶段。
- 横轴：`reward`（0/1）。
- 纵轴：出现次数（count）。
- 解读：奖励的分布直方图。比如 0 的柱子很高，说明多数动作没有获得正奖励；1 的柱子高则相反。

## 备注
- 所有图标题中会标注 `R` 值与分布名称，方便对比不同实验配置。
- 若某张图显示 “No data”，表示对应阶段在日志中没有可用数据。需要先确认 `rl_training.csv` / `rl_trace.csv` 是否写入完整。
